This page is useful reference if you're looking to find out how you should write a particular style with emotion.

## Why String Styles?

String styles are very similar to normal CSS, SCSS and etc. If you or your team is very used to writing with CSS syntax you might prefer to write with string styles.

## Why Object Styles?

Instead of writing css properties in `kebab-case` like regular css, you write them in `camelCase`, for example `background-color` would be `backgroundColor`. Object styles are especially useful with the css prop because you don't need a css call like with string styles but object styles can also be used with styled.

### Numbers

When numbers are the value of a css property, `px` is appended to the number unless it is a css property that is unitless.

```jsx
// @live
/** @jsx jsx */
import { jsx } from '@emotion/core'

render(
  <div
    css={{
      padding: 8,
      zIndex: 200
    }}
  >
    This has 8px of padding and a z-index of 200.
  </div>
)
```

### Arrays

Nested arrays are flattened.

```jsx
// @live
/** @jsx jsx */
import { jsx } from '@emotion/core'

render(
  <div
    css={[
      { color: 'darkorchid' },
      { backgroundColor: 'hotpink' },
      { padding: 8 }
    ]}
  >
    This is darkorchid with a hotpink background and 8px of padding.
  </div>
)
```

</ConditionalFromQuestion>

### Fallbacks

<ConditionalFromQuestion question="syntax" answer="string">

```jsx
// @live
/** @jsx jsx */
import { css, jsx } from '@emotion/core'

render(
  <div
    css={css`
      background: red;
      background: linear-gradient(#e66465, #9198e5);
      height: 100px;
    `}
  >
    This has a gradient background in browsers that support gradients and is red
    in browsers that don't support gradients
  </div>
)
```

</ConditionalFromQuestion>

<ConditionalFromQuestion question="syntax" answer="object">

Define fallback values for browsers that don't support features with arrays.

```jsx
// @live
/** @jsx jsx */
import { jsx } from '@emotion/core'

render(
  <div
    css={{
      background: ['red', 'linear-gradient(#e66465, #9198e5)'],
      height: 100
    }}
  >
    This has a gradient background in browsers that support gradients and is red
    in browsers that don't support gradients
  </div>
)
```

</ConditionalFromQuestion>

### With `css`

You can also use `css` with object styles.

```jsx
// @live
/** @jsx jsx */
import { jsx, css } from '@emotion/core'

const hotpink = css({
  color: 'hotpink'
})

render(
  <div>
    <p css={hotpink}>This is hotpink</p>
  </div>
)
```

</ConditionalFromQuestion>

### Composition

[Learn more composition in Emotion](/docs/composition.md).

<ConditionalFromQuestion question="syntax" answer="string">

```jsx
// @live
/** @jsx jsx */
import { jsx, css } from '@emotion/core'

const hotpink = css`
  color: hotpink;
`

const hotpinkHoverOrFocus = css`
  :hover,
  :focus {
    ${hotpink}
  }
`

const hotpinkWithBlackBackground = css`
  background-color: black;
  color: green;
  ${hotpink}
`

render(
  <div>
    <p css={hotpink}>This is hotpink</p>
    <button css={hotpinkHoverOrFocus}>This is hotpink on hover or focus</button>
    <p css={hotpinkWithBlackBackground}>
      This has a black background and is hotpink. Try moving where hotpink is in
      the css call and see if the color changes.
    </p>
  </div>
)
```

</ConditionalFromQuestion>
